# Workflow Rules

## Implementation

- 사용하지 않는 코드는 함께 제거한다.
- 변경 범위를 작고 명확하게 유지한다.
- 변경에 맞는 테스트를 실행한 뒤 결과를 확인한다.
- 주석 점검은 `app/`에 한정하지 않고 변경 범위의 모든 파일에 적용한다.
- 주석은 코드의 이름과 구조만으로 설명하기 어려운 제약, 의사결정 이유, 비직관적인 동작에만 작성한다.
- 코드가 무엇을 하는지 반복하는 주석, 구획을 나누기 위한 주석, 주석 처리된 코드는 남기지 않는다.

## Git Commit

- 커밋 타입은 `feat`, `fix`, `chore`만 사용한다.
  - `feat`: 새로운 기능
  - `fix`: 버그 수정
  - `chore`: 문서, 설정, 스타일 등 기타 변경

## Redmine

- 프로젝트는 Console(ID: 8)을 사용한다.
